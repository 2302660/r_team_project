1 Introduction

1.1 Background and Significance

The surge in digital data transmission has amplified the susceptibility of information systems to security breaches. Our project utilizes the dataset from Information is Beautiful on the world’s biggest data breaches to augment the existing visualization.

1.2 Objectives

We plan to add features to the visualization, normalize the data and run comparative analyses, and develop models to predict future breaches. Our objective is to transform this visualization into a more dynamic and analytical tool that not only reflects previous data breaches but also provides insights into potential future trends.

1.3 Data Source

The dataset is obtained from Information is Beautiful, which contains information on the world’s biggest data breaches. The dataset includes the date of the breach, the organization involved, the method of the breach, the number of records lost, and the sector of the organization.

2 Original Visualization Assessment

2.1 Description of the Visualization

The original visualization, shown in Figure 1, is a compelling graphical representation that uses varying bubble sizes and colours to depict the magnitude and frequency of major data breaches impacting organizations worldwide. Organized chronologically and segmented by year, the visualization offers a clear timeline of data breaches, providing immediate visual insight into the evolution and escalation of data security issues over time.

2.2 Critical Assessment

The original visualization excels in several areas but also presents certain limitations that could be addressed to enhance its utility and interpretability:

Strengths

  • Visual Impact: The use of large, coloured bubbles immediately draws attention and effectively communicates the relative scale of data breaches.

  • Interactivity: Interactive elements allow users to filter breaches by year and type, facilitating a customized exploration of the data.

  • Information Density: A significant amount of information is compactly displayed, providing quick insights into data breaches across different industries and regions.

Weaknesses

  • Complexity and Clarity: While the visualization holds a wealth of information, its complexity can be overwhelming at first glance, potentially confusing users unfamiliar with interpreting dense graphical data.

  • Contextual Depth: The visualization lacks contextual layers that could provide deeper insights, such as the financial impact of breaches, the nature of data compromised, or the aftermath in terms of regulatory actions and recovery.

  • Predictive Insights: There is an absence of analytical features that could predict trends or identify patterns beyond the descriptive data presented. This limits the visualizations utility for forecasting and strategic planning.

3 Proposed Enhancements

1. Normalization Features

Incorporating normalization features to compare breaches based on the size of the organization or the number of users impacted could provide a more nuanced understanding of each incident’s significance.

2. Trend Analysis

Adding trend lines or heat maps to visualize the frequency and severity of breaches over time would aid in identifying underlying patterns and potential vulnerabilities in data security practices.

3. Enhanced Interactivity

Expanding the interactivity to include more customizable filters, such as by sector or breach method, could allow users to derive more specific insights tailored to their interests or concerns.

These enhancements would not only address the current limitations but also significantly augment the visualizations value as a tool for comprehensive risk assessment and strategic planning in cybersecurity.

3. Data Transformation: We standardize and transform data to enhance its structure and usability. This includes converting data types and normalizing records lost by organizational size, if applicable.

# Convert 'year' to a numeric type
breaches_clean <- breaches_clean |>
  mutate(year = as.integer(year))

# Categorize 'Breach Type'
breaches_clean <- breaches_clean |>
  mutate(method = factor(method))

4. Augmenting Data: We may also add new variables for a more detailed analysis, such as categorizing the severity of breaches based on the number of records lost.

# Add a severity label based on the number of records lost
breaches_clean <- breaches_clean %>%
  mutate(Severity = case_when(
    records_lost > 1000000 ~ "Severe",
    records_lost > 500000 ~ "Moderate",
    TRUE ~ "Minor"
  ))
